# Milestones

Milestones are organized by delivery track:

- Infrastructure
- Backend
- Frontend

Each milestone:

- Represents a cohesive system capability
- Contains a set of GitHub issues
- Must be fully complete before moving forward

## Naming Convention

- I* → Infrastructure
- B* → Backend
- F* → Frontend

Example:

- I1 — Monorepo Setup
- B2 — Domain Models
- F3 — Event Experience

## Definition of Done

Each milestone must include:

- Implementation complete
- Tests added (unit/integration)
- API contracts defined (if applicable)
- Documentation updated
- CI passing
